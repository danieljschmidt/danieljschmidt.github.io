The BVARF toolbox for forecasting with Bayesian VARs

The github repository for the BVARF toolbox is https://github.com/danieljschmidt/BVARF.
This is a short tutorial on how to use the toolbox. If you are not familiar with Bayesian VAR models, see Banbura et al. (2010) or the handbook chapter by Karlsson.

A basic example: Forecasting US inflation using a Bayesian VAR with a Normal-Wishart prior

The function load_data loads the data from the Excel file SW_Updated.xlsx, transforms some of the time series and returns a timetable object.
data = load_data();
data = data(:, ~ismember(data.Properties.VariableNames, {'NONBORRES'}));
% exclude NONBORRES because NONBORRES < 0 during financial crisis
I want to conduct a forecast experiment for CPI inflation as target variable with a forecast horizon up to 24 months. The sample for evaluating the forecast performance is January 2000 to December 2016. I follow Banbura et al. (2010) and use a rolling estimation window of length 10 years in order to account for the possibly time-varying nature of the data generating process.
setup_r.target = 'CPIAUCSL';
setup_r.hmax = 24;
setup_r.fcst_sample = datetime(2000, 1, 1) + calmonths(0:17*12-1); % Jan 2000 - Dec 2016
setup_r.est_window = 'rolling';
setup_r.est_window_size = 120; % 10 years
First, I use an AR(1) model that will serve as a univariate benchmark.
results = containers.Map();
results('AR(1)') = run_experiment(setup_r, VAR(1,1), data(:,'CPIAUCSL'));
Then, I apply a Bayesian VAR model with 12 lags and a conjugate Normal-Wishart prior to the large dataset. For simplicity, I do not use a fully Bayesian approach to forecasting here but only use the posterior mean of the coefficients in order to generate point forecasts. Setting the shrinkage intensity turns out to be a good choice. The parameter determines the shrinkage intensity of the intercept and is thus set to a very large number. The third hyperparameter, the random walk indicator rw_ind, is set to a vector of zeros such that we shrink towards a white noise process (and not towards a random walk).
% large VAR with fixed lambda
hyperpar.theta2 = 1e6;
hyperpar.rw_ind = zeros(1,size(data,2));
hyperpar.lambda = 0.05;
bvarL = BVAR_NormalWishart(size(data,2),12,hyperpar);
results('BVAR L NW') = run_experiment(setup_r, bvarL, data, 'posteriormean');
The mean square errors relative to the AR(1) benchmark (for forecast horizons ) are summarized in the following table:
relative_msfe_table(results, results('AR(1)'), [1 3 6 12 24], 'BVAR L NW')

The BVAR forecasts for are slightly better than the AR(1) forecasts but I do not find an improvement at larger forecast horizons.

Some advanced features of the toolbox

However, the above example does not explain how to find a suitable shrinkage intensity λ. This can be done by using the empirical Bayes approach, i.e. to maximize the marginal likelihood with respect to λ. Since such an approach is computationally costly, I only use it in combination with a medium-sized dataset that does only contain a subset of the variables.
dataM = data(:, ismember(data.Properties.VariableNames, ...
{'DPCERA3M086SBEA', 'PCEPI', 'W875RX1', 'INDPRO', 'MCUMFN', ...
'UNRATE', 'PAYEMS', 'HOUST', 'GS10', 'WPSFD49207', 'PPIACO', ...
'CPIAUCSL', 'CES0600000008', 'SP500', 'M1SL', 'M2SL', 'TOTRESNS', ...
'WTISPLC', 'FEDFUNDS', 'NNUSBIS'}));
% medium-sized VAR with fixed lambda
hyperpar.theta2 = 1e6;
hyperpar.rw_ind = zeros(1,size(dataM,2));
hyperpar.lambda = 0.1;
bvarNW = BVAR_NormalWishart(size(dataM,2),12,hyperpar);
results('BVAR M NW') = run_experiment(setup_r, bvarNW, dataM, 'posteriormean');
% medium-sized VAR where lambda is determined by empirical Bayes approach
hyperpar.lambda = 'empiricalbayes';
bvarNW2 = BVAR_NormalWishart(size(dataM,2),12,hyperpar);

Since the shrinkage intensity seems to be a suitable choice for medium-sized dataset, the (computationally costly) empirical Bayes approach does not improve upon such a model. Choosing the shrinkage intensity in each time step based on an optimization problem might induce some unnecessary variation in the coefficient estimates that hampers an improvement in forecast precision. A compromise is to set the shrinkage intensity once with the empirical Bayes approach based on a pre-sample.
Furthermore, the conjugate Normal-Wishart prior imposes a restrictive structure on the prior variance of the VAR coefficients. In the so-called original Minnesota prior on the other hand, it is possible to shrink off-diagonal VAR coefficients stronger than diagonal ones by setting :
% medium-sized VAR with original Minnesota prior and fixed lambda and theta1
hyperpar.theta2 = 1e6;
hyperpar.rw_ind = zeros(1,size(dataM,2));
hyperpar.lambda = 0.2;
hyperpar.theta1 = 0.3;
bvarOM = BVAR_NormalWishart(size(dataM,2),12,hyperpar);
results('BVAR M OM') = run_experiment(setup_r, bvarOM, dataM, 'posteriormean');
relative_msfe_table(results, results('AR(1)'), [1 3 6 12 24], ...
{'BVAR L NW', 'BVAR M NW', 'BVAR M NW empBayes', 'BVAR M OM'})

Up to now we have only considered point forecasts with the posterior mean approach that does not take into account the uncertainty around the coefficient estimates. Therefore, I will conclude this tutorial by generating density forecasts:
mc_settings.ndraws = 1e3;
today= datetime(2008,10,1);
obs = dataM{today-calmonths(120):today,:};
[mcf, post] = montecarlo_forecast(bvarNW, obs, 24, mc_settings);
filter_target = strcmp(dataM.Properties.VariableNames,'CPIAUCSL');
mcf = squeeze(mcf(:,filter_target,:));
lower = quantile(mcf, 0.10, 2);
median = quantile(mcf, 0.50, 2);
upper = quantile(mcf, 0.90, 2);
figure
plot(1:24, median, 'b-')
hold on
plot(-24:0, obs(end-24:end,filter_target), 'b-')
plot(1:24, lower, 'b--')
plot(1:24, upper, 'b--')
ylabel('inflation forecast (mean +- 1 std)')
xlabel('forecast horizon')
